# 디자인 업무로그 🎨 (디자이너님께)

> 앱 실행 방법은 [설치가이드.md](./설치가이드.md), 전체 기획은 [stamp-diary-prompt.md](./stamp-diary-prompt.md) 참고.
> 지금 앱에 들어간 디자인은 전부 **개발하면서 임시로 잡은 것**이라, 뒤집어도 전혀 상관없어. 편하게 의견 줘!

## 앱 컨셉 (이것만 지켜지면 됨)

- 실물 **"Punch Stamp Notebook"**의 디지털화 — 하늘색 노트에 빈 우표 칸(톱니 테두리)이 그리드로 깔려 있고, 하루에 한 칸씩 사진 우표로 채워가는 앱
- 핵심 감성: **아날로그 우표 수집의 아기자기함**, 채워지는 재미, 빈 칸이 주는 동기부여
- 전체적으로 파스텔, 여백 넉넉, 장식 최소

## 현재 임시 디자인 값

| 요소 | 현재 값 | 상태 |
|------|---------|------|
| 배경 | 하늘색 `#A8D8EA` | 레퍼런스 노트 색 따라감 |
| 빈 우표 칸 | 회색빛 흰색 `#F0F0F0` | 레퍼런스 따라감 |
| 오늘 칸 강조 | 주황 점선 테두리 `#FF8A65` | 임시. 좀 투박함 |
| 포인트/버튼 색 | 같은 주황 `#FF8A65` | 임시 |
| 텍스트 | 진회색 `#2C3E50` / 회색 `#6B7A8C` | 임시 |
| 폰트 | 시스템 기본 | 기획상 제목은 세리프 계열 희망 (노트 표지 느낌) |
| 그리드 | 5열, 우표 비율 4:5 | 4열 vs 5열 미정 |

## 봐줬으면 하는 것들

### 1. 전체 색 팔레트 ⭐
- 지금 색 조합(하늘색+주황)이 최선인지. 포인트 컬러, 텍스트 컬러까지 hex로 제안해주면 바로 반영 가능
- 라이트 모드만 (다크모드는 v1 제외)

### 2. 앱 아이콘 + 스플래시 화면 ⭐
- 아직 Expo 기본 아이콘 그대로임. 출시 전 필수
- 앱 이름도 미정 (가칭 Stamp-a-Day) — 네이밍 아이디어도 환영

### 3. 톱니 우표 프레임 디테일
- 앱의 정체성인 컴포넌트. 톱니 크기/간격/비율이 지금 자연스러운지 (코드로 조절 가능하니 "톱니 더 촘촘하게" 같은 피드백이면 충분)
- 우표에 그림자, 테두리, 소인(우체국 도장) 같은 장식을 더할지

### 4. 월별 보드 (메인 화면)
- 4열 vs 5열 어느 쪽이 예쁜지 (기획서의 미결 사항)
- 오늘 칸 강조를 점선 말고 어떻게 할지
- 채워진 칸의 날짜 표시 방식 (지금은 우하단 흰 배지 — 투박함)
- 헤더(연/월 + 화살표) 레이아웃과 타이포

### 5. 스탬프 만들기 플로우
- 사진 선택 → 미리보기+메모 → "붙이기" 버튼으로 이어지는 화면들의 무드
- "붙이기" 순간의 애니메이션 감성 (기획: 우표가 착 붙는 스케일+페이드, 0.2~0.3초) — 참고할 레퍼런스 있으면 공유해줘

### 6. 첫 실행 온보딩
- 온보딩 화면을 둘지 말지부터 미정. 둔다면 문구/일러스트 방향

### 7. 로그인/계정 페이지 (v2 예정, 미리 레퍼런스만) ⭐
- v2에서 클라우드 백업용 계정(이메일/소셜 로그인)이 들어갈 예정
- 이 앱 감성에 어울리는 로그인 화면 레퍼런스를 미리 모아주면 좋겠어 (우표/편지/아날로그 문구 컨셉과 어울리는 것들)
- 소셜 로그인 버튼(Apple/Google)이 들어가도 안 촌스러운 레이아웃

### 8. 내가 못 챙긴 것들
- 위 항목 외에도 앱 써보다가 "이건 디자이너 눈에 거슬린다" 싶은 건 뭐든 적어줘
- 알림 문구("오늘의 우표를 붙일 시간이에요 📮") 같은 마이크로카피 톤도 환영

## 전달 형식

- 피그마 링크든, 손그림 사진이든, 텍스트 목록이든 뭐든 OK
- 색은 hex 코드, 폰트는 이름만 정확히 주면 반영 빨라
- 우선순위는 ⭐ 붙은 것부터!
